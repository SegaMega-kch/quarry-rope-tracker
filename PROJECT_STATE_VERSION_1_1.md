## Важные правила логики

### Тип, длина, диаметр

При выборе типа каната длина и диаметр подставляются автоматически.

Правила:

- `Напор ЭКГ-10` -> `41м`, `45,5 мм`
- `Подъём ЭКГ-10` -> `110м`, `45,5 мм`
- `Напор ЭКГ-12К` -> `40м`, `52 мм`
- `Подъём ЭКГ-12К` -> `82м`, `52 мм`

Если где-то появляется длина `82м`, это однозначно `Подъём ЭКГ-12К`, `52 мм`.

### Блок "Под 30т краном"

В сводке есть три кнопки:

- `На вешелах`
- `На вертушках`
- `На земле`

В этих кнопках показываются канаты, которые лежат в месте:

```text
Вешала под 30т краном
```

Кнопки открывают всплывающее меню добавления 4 видов канатов.

Компонент:

```text
app/CraneQuickAdd.tsx
```

### Добавить канат

Форма упрощена:

- тип каната
- количество
- место
- размещение
- комментарий

Длина и диаметр скрыты и подставляются автоматически.

Для мест `ЭКГ...` и `ПП...` размещение принудительно:

```text
На вертушке
```

Компонент:

```text
app/AddRopeForm.tsx
```

### Переместить канат

Форма перемещения показывает только канаты, которые есть в остатках.

Нельзя переместить больше, чем есть.

Пункт "Новое размещение" убран.

Автоматическое размещение при перемещении:

- если новое место категория `excavator` или `transfer_point` -> `На вертушке`
- если новое место `Вешала под 30т краном` -> `На вешалах`
- иначе -> `На земле`

Компонент:

```text
app/MoveRopeForm.tsx
```

### Установить на экскаватор

Кнопка установки больше не хранит отдельный остаток со статусом `Установлен`.

Устанавливать можно только канаты:

- статус `В наличии`
- место категории `excavator`
- размещение `На вертушке`

На карточке такого каната есть кнопка:

```text
УСТ
```

По нажатию:

- действие пишется в историю как `INSTALL`
- канат переводится в статус `Б/у лежит у экскаватора`
- размещение становится `На земле`
- место остается тот же экскаватор
- комментарий: `замена выполнена, б/у оставлен под экскаватором`

Идея: установленный новый канат отдельно не учитываем, потому что на каждом экскаваторе по умолчанию канат установлен. После замены важно видеть именно б/у канат, который лежит под экскаватором.

Компонент формы:

```text
app/InstallRopeForm.tsx
```

Быстрая кнопка находится в карточке остатков в:

```text
app/page.tsx
```

### Б/у канат

Форма б/у каната тоже упрощена:

- тип каната
- количество
- экскаватор
- комментарий

Длина и диаметр подставляются автоматически.

Б/у канат создается:

- место: выбранный экскаватор
- размещение: `На земле`
- статус: `Б/у лежит у экскаватора`

### Списание / вывоз б/у

Для б/у каната на карточке есть кнопка с корзиной.

По нажатию:

- канат списывается
- комментарий: `вывезен`
- физически из базы не удаляется
- из текущих карточек исчезает, потому что списанные остатки скрыты

Списывать могут:

- начальник
- кладовщик

## Карточки остатков

Карточка показывает:

- место
- количество
- тип каната коротко
- размещение
- статус, если не `В наличии`
- последнее изменение

Для б/у показывается статус `Б/у лежит у экскаватора`.

Для списанных карточки не показываются.

## Быстрые кнопки на карточках

### Корзина

Показывается для б/у канатов у экскаватора.

Действие: списать с комментарием `вывезен`.

### УСТ

Показывается для канатов у экскаватора на вертушке.

Действие: установить на этот же экскаватор.

### Перемещение с карточки

Для канатов под 30т краном на вертушке есть кнопка `⇄`.

Она открывает список мест и перемещает весь остаток с комментарием:

```text
быстрое перемещение с карточки
```

Компонент:

```text
app/CardMoveMenu.tsx
```

## Что важно помнить при будущих правках

1. Не удалять физически канаты из базы, если это рабочее действие. Лучше менять количество или статус и сохранять историю.
2. Б/у канат после вывоза списывать, а не удалять.
3. Для ЭКГ и ПП размещение в добавлении и перемещении должно быть `На вертушке`.
4. Установка доступна только для каната у экскаватора на вертушке.
5. Все действия должны писать историю в `RopeMovement`.
6. Интерфейс должен оставаться мобильным: крупные кнопки, минимум лишних полей.
